# Vibe Architecture Governance

## 适用场景
- 需要把方案文档从多版本改为单版本持续维护。
- 需要新增项目级开发约束（高内聚、低耦合、文档同更）。
- 需要在AI参与编码时建立防跑偏机制。

## 执行步骤
1. 统一文档命名：去掉版本号，保留唯一方案文档。
2. 更新项目记忆：补充架构原则和文档治理规则。
3. 建立架构文档：定义分层、依赖方向、模块边界、质量门禁。
4. 建立实施计划：按阶段交付、验收标准、风险与回滚策略。
5. 建立评审清单：强制检查跨层依赖、未测合并、无文档变更等问题。

## 强约束清单
- 高内聚、低耦合。
- 先接口契约，后实现代码。
- 单任务单主模块，跨模块改动必须拆分。
- 所有改动必须可测试、可回滚、可追踪。
- 模块完成后必须更新 files 目录文档。
