# Workspace Save Feedback Gate

## 适用场景
- 需要把“保存命令”从状态更新升级为可观测结果（成功/失败）。
- 需要在 UI 显示保存成功/失败提示，避免静默失败。
- 需要保证改动后测试和门禁稳定通过。

## 执行步骤
1. 在 Workspace 层新增保存结果模型（含 succeeded、code、message、tab）。
2. 升级 `IWorkspaceService.SaveDocument` 接收内容参数并返回结果模型。
3. 在实现层执行真实文件写入，补齐失败分支（如 `invalid_document_id`、`document_not_found`、`io_error`）。
4. 在 ViewModel 保存时传入当前草稿内容，并消费结果模型设置 UI 反馈字段（消息+错误态）。
5. 打开活动文档时加载真实文件内容到编辑区，并用状态位避免加载过程误触发脏标记。
6. 在视图层绑定反馈字段，显示成功/失败提示。
7. 为自动保存调度器增加防抖窗口与批处理出队能力（如 `DrainReady(maxBatchSize)`）。
8. 在 UI 增加“最后保存状态”和“最后保存时间”展示。
9. 在 Workspace 层接入 `FileSystemWatcher`，对外部文件变更做节流刷新。
10. 外部文件变更与本地脏编辑冲突时，必须保留本地草稿并提示冲突，不得静默覆盖。
11. 提供冲突处置动作：
	- 保留本地并覆盖保存。
	- 丢弃本地并重载外部。
12. 记录冲突审计事件（检测/处置/失败），并在 UI 暴露最新事件提示。
13. 在 ViewModel 维护最近 N 条冲突事件（建议 5 条）和展开状态，支持冲突日志面板展开/收起。
14. 在视图层增加“展开最近冲突日志”入口和事件列表面板，默认收起，按时间倒序展示。
15. 增加冲突日志范围切换：支持“当前文档过滤”与“全部文档”双视图，并在全局模式下显示文档标识。
16. 增加事件类型筛选：支持“全部/检测/处置/失败”循环筛选，并为事件类型提供一致的颜色映射。
17. 提供“一键重置筛选”能力，恢复默认筛选组合（当前文档范围 + 全部事件类型），并在会话内保持用户筛选状态。
18. 将筛选配置持久化到 `.muse/settings/conflict-log.json`，并在启动时恢复；加载/保存异常时静默降级，不影响编辑主链路。
19. 持久化写入增加防抖与最小写入间隔限制，减少高频筛选操作下的磁盘写入次数。
20. 补充测试：成功保存清脏、真实落盘写入、保存失败路径、防抖与批处理、文件监听刷新、冲突处理、冲突处置动作、冲突审计、冲突日志面板展开/收起、范围切换过滤行为、类型筛选循环与颜色映射、会话保持与重置筛选、跨实例恢复、高频切换防抖恢复、UI 反馈断言。
21. 运行门禁：build + 边界检查 + 分层测试 + 冒烟构建。

## 通过标准
- `SaveDocument` 必须返回统一结果对象。
- `SaveDocument` 必须接收待保存内容并执行真实落盘。
- 自动保存调度必须支持防抖与批量处理，避免高频编辑触发写入风暴。
- 外部文件变更必须通过节流刷新同步到工作区，且不能覆盖脏编辑。
- 冲突处理必须以“保留本地草稿优先”为默认策略，除非用户明确选择重载。
- 失败路径必须有稳定错误码和可读错误信息。
- UI 必须可见保存反馈，不允许失败静默。
- 冲突日志面板必须按时间倒序截断到最近 N 条，避免无限增长导致 UI 噪声。
- 冲突日志范围切换时必须即时刷新列表与最新提示，避免显示旧范围残留数据。
- 事件类型筛选切换时必须与范围过滤联动生效，避免筛选条件叠加错误。
- 重置筛选必须同时重置“范围+类型”两维条件，避免只重置其中一维导致用户认知偏差。
- 相关测试全部通过，质量门禁通过。

## 常见回归点
- 修改接口后未同步 Fake/Mock 导致测试编译失败。
- 打开文档不加载真实内容，导致保存覆盖为默认草稿。
- 调度队列只去重不防抖，导致高频输入时触发过多保存。
- 文件监听直接覆盖正在编辑的脏草稿，导致用户输入丢失。
- 保存失败时误清脏，导致 UI 状态与真实结果不一致。
- 成功/失败反馈未在编辑后清理，出现旧提示残留。
- 冲突日志条目未截断或排序错误，导致面板信息陈旧或过载。
- 当前文档无日志但全局有日志时，若缺少范围切换入口会造成“看不到日志”的误判。
- 事件类型颜色与文本标签不一致会误导处置判断，必须通过测试校验映射关系。
- 状态刷新后若筛选条件被意外重置，会破坏会话连续性，需要用联调测试防回归。
- 引入磁盘持久化后，测试若共用固定路径会相互污染；需在测试环境默认关闭持久化或使用独立临时目录。
- 防抖持久化是异步写入，测试应等待配置内容达成预期值后再断言恢复，避免时序抖动导致误报。
