# 回滚说明模板

## 使用说明
- 当变更可能影响主流程、架构边界或公共契约时，必须准备回滚说明。
- 回滚说明必须能够指导在最短时间内恢复到可用状态。
- 不允许只写“git revert 即可”，必须明确回滚边界与验证步骤。

## 填写示例
- 关联任务ID：S1-004
- 变更标题：三模式路由骨架
- 主模块：Muse
- 回滚负责人：待指定
- 功能触发条件：模式切换导致主界面无法正常显示。
- 性能触发条件：切换模式出现明显卡死或输入阻塞。
- 稳定性触发条件：桌面项目构建失败或启动崩溃。
- 架构违规触发条件：路由层直接耦合 Rendering 业务实现。
- 需要回滚的文件/模块：Muse/ViewModels/MainViewModel.cs、Muse/Views/MainView.axaml。
- 不应回滚的内容：Core 与 Rendering 已稳定的契约工程。
- 是否涉及配置或数据回退：否。
- 构建命令：dotnet build Muse.Desktop/Muse.Desktop.csproj
- 测试命令：dotnet test Muse.Editor.Core.Tests/Muse.Editor.Core.Tests.csproj；dotnet test Muse.Editor.Rendering.Tests/Muse.Editor.Rendering.Tests.csproj
- 门禁命令：scripts/Invoke-QualityGate.ps1
- 预期结果：回滚后桌面构建恢复通过，门禁通过。
- 技术债记录：需要重新设计更稳定的模式状态模型。
- 二次修复入口：拆分为独立状态对象后重新接入 UI。
- 文档更新要求：同步更新 files/实施计划.md 和对应变更说明。
