# 变更说明：建立渲染映射层并修正 MainView 表格渲染

- 标题：建立渲染映射层并修正 MainView 表格渲染
- 日期：2026-06-01
- 关联任务ID：S2-002
- 主模块：Muse（UI 宿主层）
- 变更类型：功能 / 修复 / 文档

## 变更内容
- 背景：为保证 UI 与 Rendering 层解耦，需提供一个映射层把 `RenderingResult` 转为 UI 可消费的轻量 ViewState；同时修复预览中 Markdown 表格未正确渲染的问题。
- 本次改动：
  - 新增 `IMarkdownPreviewService` / `MarkdownPreviewService`（映射 RenderingResult -> Preview ViewState）。
  - 在 `Views/MainView.axaml` 中新增 `PreviewTableRowTemplate` DataTemplate，渲染 `PreviewTableRowViewModel` 与其 `CellViewModels`。
  - 修正 `Muse.ThemeUX/ThemeResources.xaml`：将 `IconFontFamily` 由 `x:String` 改为 `FontFamily`，避免运行期类型转换。
  - 移除 `MainView` 中遗留的本地样式，样式已迁移到 `Muse.ThemeUX/ThemeStyles.xaml`（样式文件已存在）。
  - 添加/更新对应的测试（`Muse.Tests` 与相关模块测试），确保映射层行为可被覆盖。

- 未改动内容：解析器实现（Rendering 层内部细节）未变化；Rendering DTO 与适配层契约保持兼容。
- 为什么现在做：S2-002 为用例链路的关键步骤，需先保证 UI 只消费 DTO，且预览表格渲染问题影响用户体验，优先修复并覆盖测试。

## 影响分析
- 影响模块：Muse、Muse.ThemeUX、Muse.Tests
- 对外行为变化：UI 渲染来源由占位文本替换为映射后的 Preview ViewState，表格/代码块等结构化渲染在预览区正确显示。
- 兼容性影响：向后兼容，RenderingResult contract 未改变。
- 性能影响：映射层为轻量同步映射，对现有延迟影响微小（如需可延迟到后台线程优化）。
- 风险点：映射逻辑若复杂化可能导致与增量渲染对接变重，建议保持映射层仅做视图级转换。
