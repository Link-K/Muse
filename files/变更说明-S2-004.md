# 变更说明：S2-004 渲染块样式与预览交互

- 任务 ID：S2-004
- 任务名称：渲染块样式与预览交互（块卡片、代码块高亮、表格对齐、复制/锚点交互）
- 负责人：待分配
- 主模块：Muse（UI） / Muse.Editor.Rendering（渲染 DTO）
- 当前状态：进行中

## 变更目的
完善预览区的渲染块交互体验，使预览不仅展示渲染结果，还支持常用操作：
- 从代码块一键复制代码（Copy）
- 复制 heading 的锚点链接（Copy Anchor）
- 表格对齐与可替代纯文本展示
- 代码块语言显示与语法高亮准备（后续集成高亮库）

## 实施内容（分步）
1. 实现 `PreviewBlockViewModel.CopyCodeCommand`：调用 Avalonia 剪贴板 API，将 `Content` 写入剪贴板；在测试环境提供可替换的 `IClipboardService` 以便单元测试模拟。
2. 实现 `PreviewBlockViewModel.CopyAnchorCommand`：基于 Heading 文本生成锚点（slugify），并拷贝到剪贴板；同步显示短暂反馈（`SaveFeedbackMessage` 或局部提示）。
3. 表格样式完善：在 `PreviewTableRowTemplate` 中优化列间距与对齐，加入等宽字体或按需回退纯文本显示。已有模板已加入 `PreviewTableRowTemplate`，需在样式中保证边框与间隔一致。
4. 代码块交互：提供语言显示（已在 `PreviewBlockViewModel.CodeFenceLanguage` 中），后续接入语法高亮库（可选，S2-004 后半段）。
5. 单元测试：为复制命令添加单元测试（通过注入 `IClipboardService` 测试复制行为），并做手动冒烟测试验证 UI 交互（在本地桌面运行）。

## 测试场景
- 代码块复制：将不同长度与包含换行的代码复制到剪贴板，校验剪贴板内容一致。
- 标题锚点复制：不同字符集标题（含空格/特殊符号）生成的锚点符合 slug 规则。
- 表格展示：对齐与分隔线在宽窄窗口下显示稳定。
- 无头 CI 兼容：确保单元测试不依赖真实系统剪贴板（通过 `IClipboardService` mock）。

## 验收标准
- `CopyCodeCommand` 与 `CopyAnchorCommand` 的单元测试通过。
- 在本地 `Muse.Desktop` 中功能可用并做手动验证（代码复制、锚点复制、表格显示）。
- 相关文档已更新（本文件与 `files/实施计划.md`）。
