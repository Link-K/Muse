# 任务卡模板

## 使用说明
- 一张任务卡只允许对应一个主模块。
- 开始编码前必须先写完本卡。
- 若任务跨模块，必须拆成多张任务卡并建立依赖顺序。
- 完成任务后必须同步更新相关文档。
- 推荐配套使用：`files/开发检查清单.md`。

### 阶段检查清单（可勾选）
- [ ] 任务开始前：边界、影响范围、验收标准已明确。
- [ ] 实现中：未引入跨层耦合，改动保持小步可验证。
- [ ] 模板与绑定：DataTemplate 无不稳定跨层级绑定，复杂对象已资源化。
- [ ] 持久化与恢复：状态写入与恢复路径均覆盖正常和缺失场景。
- [ ] 异常路径：关键异常具备可定位信息与最小修复方案。
- [ ] 提测前：已执行构建与测试并记录结果。
- [ ] 文档同步：变更说明、实施计划、项目记忆已更新。

### 边缘场景回归（按需勾选）
- [ ] 快速连续操作（如双击/拖拽并发）稳定。
- [ ] 空数据或缺失文件输入稳定。
- [ ] 持久化文件损坏/缺失时可降级运行。
- [ ] 重启恢复行为符合预期。

## 填写示例
- 任务ID：S1-002
- 任务名称：抽象编辑器核心接口契约
- 主模块：Muse.Editor.Core
- 关联阶段/Sprint：Sprint 1
- 前置依赖：S1-001
- 需求目标：定义最小可测试的核心接口契约。
- 不做范围：不落具体编辑实现，不接入 UI。
- 输入输出契约：输入为编辑器核心行为需求，输出为会话/命令/文档模型接口。
- 影响范围：Core 契约、测试工程。
- 架构约束：禁止引入 UI 类型与基础设施依赖。
- 禁止事项：禁止顺手改动 Rendering 或 UI 模块。
- 依赖新增：无。
- 是否允许改动公共契约：是，仅限 Core 契约。
- 风险点：接口过宽导致实现耦合。
- 回滚条件：接口设计引入 UI 或基础设施依赖。
- 回滚方案：回退到最小接口集版本。
- 测试用例：契约执行、撤销重做、异常路径。
- 验收标准：契约可被测试驱动，无越层依赖。
- 需要更新的文档：files/实施计划.md、files/项目架构文档.md。
- 完成后执行的门禁命令：scripts/Invoke-QualityGate.ps1
