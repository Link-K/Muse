# Muse 项目实施计划（Vibe Coding执行版）

## 1. 计划目标
- 在保证架构稳定和可维护的前提下，快速交付类Typora Markdown编辑器MVP。
- 通过标准化任务模板与质量门禁，减少AI生成代码带来的漂移与返工。

## 2. 迭代节奏
- 节奏：双周迭代（2周一个Sprint）。
- 每周结构：
  - 周一：需求澄清与任务拆解。
  - 周二至周四：开发与联调。
  - 周五：回归、文档更新、里程碑复盘。

## 3. 阶段规划
## 阶段A（Sprint 1）：骨架与边界落地
### 目标
- 建立分层架构与模块边界。
- 建立Vibe Coding任务模板与质量门禁。
### 交付物
- 模块工程骨架与依赖约束。
- 编辑模式/分屏模式/阅读模式基础路由。
- 文档模板：任务卡、变更说明、回滚说明。
### 验收
- 架构检查通过，无跨层反向依赖。
- 空跑流程可完成：任务->实现->测试->文档更新。

## 阶段B（Sprint 2-3）：编辑核心能力
### 目标
- 打通所见即所得主链路，覆盖高频Markdown语法。
### 交付物
- 编辑模式默认体验（实时渲染）。
- 分屏模式（左右/上下）联动滚动与定位。
- 阅读模式只读渲染。
- 基础撤销/重做与自动保存。
### 验收
- 高频语法输入行为与Typora接近。
- 连续切换模式无明显闪烁与状态丢失。

## 阶段C（Sprint 4）：工作区与资源管理
### 目标
- 完成本地文件树、多标签、资源落盘策略。
### 交付物
- 文件树（增删改查）、多标签、脏标识。
- 图片拖拽/粘贴落地至同级 assets。
- 崩溃恢复与会话恢复。
### 验收
- 资源路径相对化正确，项目迁移后可用。
- 异常退出后可恢复最近会话。

## 阶段D（Sprint 5）：导出能力
### 目标
- 完成HTML/PDF/Docx导出及菜单化高级选项。
### 交付物
- 导出菜单可配置选项。
- PDF高级选项：目录页、页眉页脚、水印（按需勾选）。
### 验收
- 同一文档多格式导出结构一致。
- 高级选项生效准确、组合行为可预测。

## 阶段E（Sprint 6）：主题、动效与体验打磨
### 目标
- 落地apple-design视觉规范与动效策略。
### 交付物
- 液态玻璃浅色/深色默认主题。
- Typora CSS主题导入兼容。
- 光标平滑移动、淡入淡出、平滑切换、三点闪烁加载反馈。
### 验收
- 多主题下可读性达标。
- 动效不影响输入响应。

## 阶段F（Sprint 7）：AI与RAG
### 目标
- 打通本地与云模型双路选择及写作辅助。
### 交付物
- 模型选择开关（设置菜单）。
- 检索索引、问答、改写/总结回写。
### 验收
- 模型切换稳定可用，失败路径有清晰提示。
- 关闭AI后核心编辑能力不受影响。

## 阶段G（Sprint 8）：性能与发布准备
### 目标
- 达成启动时间小于1秒与核心稳定性指标。
### 交付物
- 启动优化、加载过渡界面。
- 回归测试与发布清单。
### 验收
- 启动耗时达标。
- 大文档与高频切换场景无明显卡顿。

## 4. Vibe Coding任务模板（每个任务必填）
- 任务名称。
- 所属模块（仅一个主模块）。
- 需求背景与目标。
- 输入输出契约。
- 风险与回滚方案。
- 测试用例与验收标准。
- 文档更新点（必须填写）。

## 5. 研发协作规则
- 小步提交：每次改动可独立验证。
- 禁止混合提交：功能、重构、格式化分开。
- 先测后并：测试未通过不得合并。
- 文档同更：代码合并前必须更新 files 目录文档。

## 6. 度量指标（每个Sprint复盘）
- 质量：缺陷密度、回归失败率、回滚次数。
- 效率：任务周期、中断次数、返工率。
- 体验：启动耗时、编辑延迟、模式切换耗时。
- AI治理：AI生成代码一次通过率、违规次数（跨层依赖/未测合并）。

## 7. 风险与缓解
- 风险：AI一次性改动过大导致回归风险高。
  - 缓解：限制单任务改动范围与文件数量。
- 风险：需求频繁变更导致模块边界被侵蚀。
  - 缓解：先更新架构文档后改代码。
- 风险：动效过多影响输入性能。
  - 缓解：建立动效性能预算与降级开关。

## 8. 里程碑检查点
- M1：阶段B完成（可稳定编辑）。
- M2：阶段D完成（可稳定导出）。
- M3：阶段F完成（AI辅助可用）。
- M4：阶段G完成（性能达标可发布）。

## 9. 边缘场景测试清单（必须覆盖）
- 超大文档（10万字）连续编辑与切模式。
- 图片密集文档（200+图片）滚动与保存。
- 主题切换与Typora CSS导入后的可读性。
- 模型切换（本地/云）过程中的失败重试。
- 导出高级选项组合（目录页+页眉页脚+水印）。
- 异常断电或崩溃后的自动恢复。

## 10. 执行要求
- 每个阶段完成后，必须更新：
  - 项目架构文档。
  - 本实施计划状态。
  - 对应模块开发文档（files目录）。

## 11.4 Definition of Done（Sprint 1）
- 所有任务均满足主模块唯一归属，不跨任务无序扩散。
- 通过编译、单元测试、模式切换冒烟测试。
- 完成架构文档、实施计划、任务卡模板的同步更新。
- 未引入未评审第三方依赖。

## 11.5 风险预警与回滚点
- 风险：骨架阶段过早堆功能导致结构失稳。
  - 预警信号：任务出现跨3个以上模块改动。
  - 回滚点：回退到接口契约稳定版本，仅保留骨架能力。
- 风险：分屏与模式切换状态管理混乱。
  - 预警信号：切换后光标或滚动状态丢失。
  - 回滚点：先保留单一状态源，再恢复高级联动。

## 13. 首批任务卡实例（S1-001 ~ S1-003）

### 13.1 S1-001 建立解决方案分层骨架
- 任务ID：S1-001
- 任务名称：建立解决方案分层骨架
- 主模块：Muse.Shell
- 当前状态：已完成（已创建模块项目骨架并接入解决方案）
- 需求目标：在现有仓库中建立分层架构骨架与项目引用规则，确保后续功能按边界落地。
- 输入输出契约：
  - 输入：当前 `Muse.sln` 与现有项目结构。
  - 输出：分层模块目录与项目引用关系说明（UI -> Application -> Domain -> Infrastructure 抽象接口）。
- 影响范围：解决方案文件、项目引用关系、基础目录结构。
- 风险与回滚：
  - 风险：一次性重排引用导致编译失败。
  - 回滚：按提交粒度回退引用改动，仅保留目录结构。
- 测试用例：
  - 用例1：全量构建通过。
  - 用例2：故意引入反向依赖时可被规则识别。
- 验收标准：
  - 编译通过。
  - 无跨层反向依赖。
  - 分层说明文档更新完成。

  - 已执行 `dotnet build .\Muse.sln` 并通过。
  - 当前存在 3 个警告：其中 1 个来自 PowerShell 对 `;ForceNoAlign` 的命令分隔解释，不属于项目代码问题；其余 2 个为 Android 既有 `SkiaSharp.NativeAssets.Android` 16 KB 页面大小警告，与本次骨架改动无关。

  - 更新 `files/项目架构文档.md` 的模块映射与依赖图说明。

### 13.2 S1-002 抽象编辑器核心接口契约
- 任务ID：S1-002
- 任务名称：抽象编辑器核心接口契约
- 主模块：Muse.Editor.Core
- 需求目标：定义可测试、可扩展的核心接口，约束后续AI生成实现不偏离领域边界。
- 输入输出契约：
  - 输入：编辑器核心行为需求（会话、命令、文档模型、撤销重做）。
  - 输出：`IEditorSession`、`ICommand`、`IDocumentModel` 等接口与错误语义约定。
- 影响范围：Core 模块接口定义、Application 层调用契约。
- 风险与回滚：
  - 风险：接口过宽导致后续实现耦合。
  - 回滚：冻结最小接口集，删除未使用接口后再扩展。
- 测试用例：
  - 用例1：会话创建与销毁生命周期测试。
  - 用例2：命令执行/撤销/重做契约测试。
  - 用例3：非法输入与异常路径测试。
- 验收标准：
  - 接口可被单元测试驱动。
  - 无 UI/基础设施类型泄漏到 Core。
  - 方法命名与参数语义明确。
- 文档更新点：
  - 更新 `files/Markdown编辑器-PRD与技术方案.md` 中核心模块接口说明。

### 13.3 S1-003 搭建渲染引擎适配层
- 任务ID：S1-003
- 任务名称：搭建渲染引擎适配层
- 主模块：Muse.Editor.Rendering
- 需求目标：完成Markdown渲染适配层桩实现，打通“输入Markdown -> 渲染结果模型”基础链路。
- 输入输出契约：
  - 输入：Markdown文本、渲染选项（主题/模式上下文）。
  - 输出：统一渲染结果对象（供编辑模式与分屏模式消费）。
- 影响范围：Rendering 模块、Shell 模式切换渲染接入点。
- 风险与回滚：
  - 风险：渲染结果模型与UI耦合过深。
  - 回滚：回退为纯领域渲染结果DTO，UI侧自行映射。
- 测试用例：
  - 用例1：标题、列表、代码块、表格的基础渲染转换。
  - 用例2：空文档与超长段落渲染稳定性。
  - 用例3：渲染异常时错误对象返回与日志记录。
- 验收标准：
  - 可输出基础渲染结果。
  - 与 UI 层解耦。
  - 为后续增量渲染保留扩展点。
- 文档更新点：
  - 更新 `files/项目架构文档.md` 中 Rendering 模块职责与边界。
