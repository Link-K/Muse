# Muse 开发检查清单（任务执行版）

日期：2026-06-02
适用范围：Muse 所有功能开发、缺陷修复与重构任务

## 1. 任务开始前
- 明确任务边界：只改一个主目标，避免混入无关重构。
- 明确影响模块：记录将改动的项目与文件范围。
- 明确验收标准：至少包含功能行为、异常路径、回归影响。
- 确认文档落点：本次需要更新的 `files/` 文档清单。

## 2. 代码实现中
- 保持高内聚低耦合：UI、ViewModel、Service 职责分离。
- 小步提交改动：每步可独立编译或验证。
- 避免不稳定绑定：DataTemplate 内优先使用项级命令代理，减少跨层级根 DataContext 依赖。
- 避免复杂内联模板对象：动画/过渡优先资源化复用。
- Avalonia 属性显式声明：涉及过渡属性时优先显式写法（如 `Visual.OpacityProperty`）。

## 3. 标签与工作区相关（S2-006经验）
- 拖拽状态放在 `WorkspaceTabViewModel`：`IsDropTarget`、`IsDropBefore`、`IsDropAfter`。
- 标签重排通过 `MainViewModel.ReorderTabs` 统一入口。
- `MoveTab` 成功后立即持久化 `.muse/settings/tabs.json`。
- 恢复顺序采用：recovery 快照优先 + tabs 顺序重排 + 缺失文件忽略。

## 4. 异常处理与诊断
- 先定位再修改：按堆栈行号精确定位 XAML/代码节点。
- 采用最小修复：优先改最小可疑点，避免扩大影响面。
- 每次修复后做双回归：
  - `dotnet build`
  - `dotnet test --no-build`
- Windows 终端 PATH 异常时使用绝对路径运行 dotnet：
  - `C:\Program Files\dotnet\dotnet.exe`

## 5. 提测前检查
- 功能路径验证：主流程至少一次手动冒烟。
- 边缘路径验证：空数据、无效输入、快速连续操作。
- 回归确认：相关测试全绿，不引入新失败。
- 文档同步完成：
  - `files/变更说明-*.md`
  - `files/实施计划.md`
  - `memory/Muse-project-memory.md`

- 多标签连续重排后重启，确认 `tabs.json` 恢复顺序正确。
- 从空标签到首个标签打开，确认指示器状态与动画稳定。
- 持久化文件损坏/缺失时，确认系统降级行为可用且不崩溃。
