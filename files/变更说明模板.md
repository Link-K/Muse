# 变更说明模板

## 使用说明
- 每次功能提交前都应有一份对应的变更说明。
- 变更说明必须和代码范围一致，禁止写成泛泛的“优化若干问题”。
- 若存在风险、兼容性影响或迁移要求，必须明确写出。

## 填写示例
- 标题：新增 Rendering 契约与适配层骨架
- 日期：2026-05-30
- 关联任务ID：S1-003
- 主模块：Muse.Editor.Rendering
- 变更类型：功能
- 背景：需要先定义纯 DTO 级渲染输出，避免 UI 与解析逻辑耦合。
- 本次改动：新增 RenderRequest、RenderedBlock、MarkdownEngineAdapter 及测试工程。
- 未改动内容：未接入 UI 映射，未接入真正 Markdown 第三方解析器。
- 为什么现在做：为编辑模式和分屏模式的渲染接入提供稳定边界。
- 影响模块：Muse.Editor.Rendering、Muse.Editor.Rendering.Tests。
- 对外行为变化：新增渲染契约接口。
- 兼容性影响：无外部 API 兼容问题。
- 性能影响：当前为骨架实现，暂不代表最终性能。
- 风险点：块分类规则目前为占位实现。
- 执行命令：dotnet test Muse.Editor.Rendering.Tests/Muse.Editor.Rendering.Tests.csproj
- 测试结果：3/3 通过。
- 构建结果：通过。
- 是否通过质量门禁：是。
- 已更新文档：files/实施计划.md、files/项目架构文档.md。
- 后续待做：接入真实 Markdown 解析与增量渲染。
